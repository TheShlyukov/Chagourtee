# План реализации поддержки медиафайлов (фото, видео, файлы, аудио)

## Введение

В этом документе описан план реализации поддержки отправки различных типов медиафайлов (фото, видео, файлы, аудио) в чат-приложении Chagourtee. Реализация будет соответствовать архитектурным требованиям проекта и обеспечивать безопасность данных.

## Требования

- Все загружаемые пользователем медиафайлы должны храниться в директории `server/data/media/`
- Файлы должны быть зашифрованы при хранении
- Поддержка различных типов файлов: изображения, видео, аудио, документы
- Интеграция с существующей системой сообщений

```bash

```

### 1.2 Создание эндпоинта загрузки

Реализовать POST `/api/upload` эндпоинт, который:
- Проверяет аутентификацию пользователя
- Проверяет тип файла на соответствие разрешенным расширениям/типам MIME
- Ограничивает размер файла (например, максимум 50 МБ)
- Шифрует содержимое файла с использованием AES-256-GCM
- Сохраняет зашифрованный файл в `server/data/media/`
- Сохраняет метаданные в новой таблице `media_files`

### 1.3 Обновления схемы базы данных

Добавить таблицу `media_files` со следующими полями:
- [id](file:///Users/daniil/Desktop/Chagourtee/client/src/api.ts#L130-L130) - первичный ключ
- `message_id` - связь с сообщением
- [original_name](file:///Users/daniil/Desktop/Chagourtee/client/src/types/messages.ts#L4-L4) - оригинальное имя файла
- `encrypted_filename` - имя зашифрованного файла

### 1.4 Создание эндпоинта для обслуживания медиафайлов

Реализовать GET `/api/media/:filename`, который:

- Читает зашифрованный файл
- Расшифровывает и передает его обратно клиенту

## Технические детали реализации

### 1. Стратегия шифрования

- Использовать AES-256-GCM для шифрования файлов
- Хранить ключи шифрования отдельно (в переменных окружения)
- Генерировать уникальные IV для каждого файла

### 2. Организация хранения

- Организовать файлы в подкаталоги по дате/месяцу для производительности
- Использовать UUID для имен зашифрованных файлов, чтобы предотвратить угадывание
- Поддерживать сопоставление в базе данных между оригинальными и зашифрованными именами

### 3. Рассмотрение вопросов производительности

- Реализовать кэширование файлов там, где это целесообразно
- Оптимизировать размеры изображений для веб-доставки
- Рассмотреть возможность отложенной загрузки для чатов с большим количеством медиафайлов

### 4. Миграция базы данных

- Добавить скрипт миграции для обновления схемы
- Обеспечить обратную совместимость с существующими текстовыми сообщениями
